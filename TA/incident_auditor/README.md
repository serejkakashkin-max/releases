# TA · Аудитор инцидентов

Источник: `C:\Users\snkashkin\Downloads\Архив`

Владелец бизнес-логики: TA / Тутов Артём.

Дата интеграции: 2026-07-16.

## Назначение

Экспериментальный web-инструмент анализа закрытия ИТ-инцидентов по загруженному Excel-файлу.

## Интеграционные изменения

- Flask-приложение преобразовано в optional Blueprint.
- Модуль подключается под `/sandbox/ta/incident-auditor`.
- Внутренние ссылки переведены на `url_for`.
- Runtime перенесён в `cache/ta_incident_auditor`.
- Интерфейс адаптирован под визуальный стиль основного проекта.

## Runtime

Рабочие файлы модуля находятся в:

```text
cache/ta_incident_auditor/
  uploads/
  exports/
  state/
```

## Включение И Отключение

Модуль включается через локальный SUP-конфиг:

```json
{
  "modules": {
    "ta_incident_auditor": {
      "enabled": true
    }
  }
}
```

Изменение флага требует перезапуска Flask, потому что Blueprint регистрируется при старте приложения.

## Удаление

Папку `TA/` можно удалить. Основной проект продолжит запускаться, а карточка TA исчезнет из Песочницы.

## Ограничения Текущей Версии

- Поддерживается только `.xlsx`.
- CSV не реализован в исходной web-версии.
- Используется глобальное состояние `incidents`, унаследованное из исходного приложения.
- Параллельные загрузки могут заменять данные друг друга.
- Нет отдельной многопользовательской модели.
- Бизнес-логика, правила анализа и дальнейшее развитие принадлежат владельцу модуля TA.
