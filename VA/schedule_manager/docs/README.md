# Документация проекта

## Источники требований

- `source/ui-and-integrations-guide.html` — исходная страница требований к UI/API/интеграциям.
- `ui-and-integrations-source.md` — краткая фиксация обязательных правил из требований.
- `duty-scheduling-rules.md` — правила формирования и проверки графиков дежурств.
- `migration-build.md` — правила сборки и подключения модуля к родительской АС.

## Документы для использования АС

- `user-guide.md` — как пользоваться веб-интерфейсом.
- `api.md` — список API и форматы запросов/ответов.
- `changelog.md` — журнал изменений по доработкам АС.

## Правило сопровождения

При каждой функциональной доработке нужно обновлять связанные документы:

- `user-guide.md`, если меняется пользовательский сценарий.
- `api.md`, если меняется или добавляется API.
- `ui-and-integrations-source.md`, если уточняется трактовка требований.
- `changelog.md`, всегда при изменении проекта.
