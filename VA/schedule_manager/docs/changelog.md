# Журнал изменений

## 2026-07-14

- Добавлена миграционная сборка для родительской АС: health-check, manifest модуля, поддержка mount-префикса и скрипт сборки архива без локального мусора, рабочих графиков, персональных данных и настроек интеграций.
- Добавлена первая версия автопланировщика: кнопка `Автоплан` доступна только на пустом графике с праздниками и запускается через обязательное модальное подтверждение внесенных отпусков.
- Уточнено условие автопланирования: месяц без праздничных дней не блокирует запуск, если график пустой; пользователь подтверждает, что проверил и отпуска, и праздники.
- После автопланирования месяц помечается как сформированный автопланировщиком: под графиком выводится дата формирования и пояснение, почему смены распределены именно так.
- В артефакт автопланировщика добавлены пояснения по конкретным назначениям: сотрудник, смена, период, доступные кандидаты, нагрузка до назначения и причина выбора.
- В таблице графика добавлены хинты автопланировщика для дежурных смен; для `8`, `Праздник` и `Отпуск` хинты не показываются.
- Автопланировщик теперь умеет дозаполнять частично подготовленный график: уже внесенные вручную смены не изменяются, а начатые недельные блоки продолжаются тем же сотрудником.
- В справочник сотрудников добавлен признак `Готов к сверхурочке`; автопланировщик назначает `ВХ` на каждый выходной или праздничный день активным московским сотрудникам с включенным признаком.
- Балансировка `ВХ` уточнена: учитываются `ВХ` за три предыдущих месяца и в текущем месяце, а руководитель допускается к `ВХ` с наименьшим приоритетом.
- Проверка графика теперь требует ровно одну `ВХ` на каждый выходной или праздничный день и проверяет переходящие недельные смены между соседними месяцами.
- Если новый месяц начинается внутри рабочей недели, автопланировщик продолжает недельные смены `ХД`/`ХР`/`ДД`/`ДР`/`ВД`/`ВР` из предыдущего сохраненного графика.
- Автопланировщик теперь учитывает нагрузку за три предыдущих месяца: дежурные смены прошлых периодов становятся стартовым весом кандидата при распределении.
- Балансировка автопланировщика уточнена: недельная смена увеличивает нагрузку на фактическую длину блока, а кандидаты сначала сравниваются по количеству дежурных блоков в текущем месяце.
- Автопланировщик больше не повторяет одну и ту же московскую дежурную смену у одного сотрудника в пределах месяца при наличии альтернатив.
- Автопланировщик разводит вечерние смены `ВД`/`ВР` между соседними неделями при наличии альтернатив, включая переход между месяцами.
- При назначении вечерних смен `ВД`/`ВР` автопланировщик теперь отдает приоритет сотрудникам с компетенцией `МПР-координатор`, но не ставит двух МПР-координаторов в вечерние смены одновременно.
- Автопланировщик теперь учитывает связку МПР: если один МПР-координатор назначен в `ДД`, другой МПР-координатор не назначается в вечернюю смену той же недели при наличии альтернатив.
- Добавлена системная компетенция `Новичок`; автопланировщик не назначает новичка в основные дежурные смены `ДД`, `ВД` и `ХД`, если в этой локации есть другие доступные сотрудники.
- При любой ручной корректировке графика через ячейку, массовое заполнение, добавление или удаление сотрудника отметка автопланировщика очищается.
- Автопланировщик работает с сохраненными данными АС, закрывает недельные дежурные смены, учитывает локации, руководителя, отпуска на неделе и назначает один `ВХ` на каждый выходной или праздничный день.
- После автопланирования график сохраняется в `schedule_data.json`, часы пересчитываются целыми числами, а результат проходит стандартную проверку правил.
- В форме сотрудника поле `Компетенция` заменено на адаптивный мультиселект: выбранные компетенции показываются чипами, внутри списка есть поиск и фильтр `Все`/`Выбранные`.
- Окно редактирования сотрудника стало лучше масштабироваться по ширине: поля перестраиваются в гибкую сетку, а выбор компетенций не раздувает форму.
- Справочник сотрудников переведен на inline-редактирование по аналогии со статусом: ФИО, компетенции, email, телефон, статус и локация сохраняются прямо из строки таблицы.
- Справочники `Компетенции` и `Смены` приведены к тому же принципу: существующие строки редактируются inline, добавление открывается через компактную кнопку `+`.
- В справочнике смен поле `Часы` больше не показывает лишнюю дробную часть: `8.0` отображается как `8`.
- В информере `Сегодня` дата теперь отображается человекочитаемо, например `14 июля`, вместо технического формата `2026-07-14`.
- В `duty-scheduling-rules.md` закреплены новые правила планирования: недельная целостность смены как обязательная ошибка и нежелательность вечерних смен две недели подряд как предупреждение.
- В правилах отдельно зафиксировано, что обязательные проверки должны выполняться детерминированным валидатором АС; ИИ может помогать с вариантами автопланирования, но не заменяет проверку.
- Форма добавления сотрудника перенесена в модальное окно, открываемое компактной кнопкой `+` у списка сотрудников.
- Счетчики `Активных` и `Всего` перенесены из крупного подзаголовка в аккуратные служебные метки рядом с заголовком `Список сотрудников`.
- Поле `Компетенция` в справочнике сотрудников вынесено ближе к ФИО и показано отдельной колонкой в каждой строке сотрудника.
- Для каждого сотрудника в рабочих данных графика добавлено явное поле `competencies`; новые импорты Excel подтягивают компетенции из справочника сотрудников или назначают базовую компетенцию `Сотрудник сопровождения`.
- В справочнике сотрудников пустой набор компетенций отображается отдельной строкой `Не задана`, чтобы отсутствие настройки было видно пользователю.
- Добавлен отдельный справочник `Компетенции` с системными компетенциями `Руководитель`, `Сотрудник сопровождения`, `МПР-координатор`.
- В карточке сотрудника одиночная роль заменена на набор компетенций; один сотрудник может иметь несколько компетенций одновременно.
- Проверка графика теперь использует компетенции: руководитель ограничен сменой `8`, МПР-координаторы не могут одновременно быть в вечерних сменах, а при дневном основном дежурстве одного МПР-координатора другой не должен быть в отпуске или вечерней смене.
- Добавлен API `GET /api/competencies` для чтения справочника компетенций.
- Списки сотрудников Москвы/Хабаровска и руководителя вынесены из `duty_rules.py` в справочник сотрудников: проверки графика теперь используют поля `Локация` и `Роль в графике`.
- В справочник сотрудников добавлены редактируемые поля `Локация` и `Роль в графике`; текущие сотрудники размечены по прежним правилам.
- В таблице графика колонка `Часы` теперь отображается целым числом без десятичной части.
- Таблица графика теперь масштабируется по ширине окна: удалена жесткая минимальная ширина, колонки дней делят доступное пространство, длинные значения обрезаются внутри ячейки.
- Область таблицы графика стала адаптироваться к высоте текущего окна вместо фиксированной высоты.
- Высота строк и заголовков таблицы графика теперь адаптируется к высоте окна; кнопки и выпадающий список внутри ячеек масштабируются вместе со строкой.
- Таблицы справочников `Сотрудники` и `Смены` теперь также адаптируются по ширине и высоте: убраны жесткие минимальные ширины, строки и элементы управления масштабируются под окно.
- В справочнике смен скрыто отдельное поле `Код`: пользователь заполняет `Сокр.`, а для новых смен оно используется как внутренний ключ.
- В справочнике сотрудников действия `Изменить` и `Удалить` оформлены как кнопки вместо текстовых ссылок.
- В справочнике смен удалена отдельная кнопка `Изменить`: параметры смены меняются прямо в таблице.
- Реализован экспорт текущего графика в Excel: книга содержит три видимых листа — сам график месяца, `К публикации` и `Для внесения в табель`, с легендами и преобразованием смен в табельные интервалы.
- В экспортируемой Excel-книге включено обязательное масштабирование текста внутри ячеек, чтобы длинные подписи не обрезались.
- В Excel-экспорте отключен перенос текста в ячейках, чтобы `shrink_to_fit` реально уменьшал шрифт под размер ячейки, включая лист `Для внесения в табель`.
- Кнопка `Экспорт` перенесена в верхний заголовок окна рядом с `Импорт`.
- Действия с графиком перенесены в раздел `График`: создание, копирование, загрузка файла, добавление сотрудника и удаление графика теперь находятся рядом с таблицей.
- Заголовок раздела `График` оптимизирован: текущий месяц показан рядом с названием, `Импорт` вынесен вправо в заголовок.
- `Импорт` перенесен в правую часть верхнего заголовка окна; в разделе `График` кнопка больше не дублируется.
- Блок `Состояние на сегодня` перестроен: основной дежурный остался отдельной плашкой, остальные смены показаны справа компактной легендой в 2 колонки; основная смена `8` скрыта.
- Панель состояния на сегодня дополнительно ужата до небольшой информационной строки с чипами смен, без отдельного крупного заголовка и карточной раскладки.
- Плашка основного дежурного перестроена в порядок `[код смены] [ФИО]`, с названием смены строкой ниже.
- Индикатор выделения и кнопка `Снять выделение` перенесены в строку выбора года и месяца.
- Блок `Состояние на сегодня` уплотнен и оставлен только как информер по текущей дате; кнопка API из него убрана.
- Фильтры года и месяца перенесены в раздел `График`, сделаны компактнее и применяются автоматически при выборе значения без кнопки `Показать`.
- Запуск локального сервера переведен на стабильный режим без Flask reloader: `run.py` по умолчанию использует порт `5055`, порт можно задать через `SCHEDULE_MANAGER_PORT`.
- `scripts/run_dev.sh` теперь проверяет занятость порта и показывает процесс, который уже слушает этот порт.
- Улучшено выделение ячеек графика: добавлено протягивание мышью прямоугольного диапазона, `Shift` выбирает диапазон от якоря, `Ctrl`/`Cmd` добавляет или снимает отдельные ячейки.
- Добавлено выделение нескольких отдельных диапазонов: `Ctrl`/`Cmd` + протягивание мышью добавляет новый диапазон к уже выделенным.
- `Escape` снимает текущее выделение ячеек.
- Выпадающий список выбора смены в ячейке сделан компактным и больше не растягивает таблицу.
- Редактор смены в ячейке теперь закрывается без сохранения при клике в другую ячейку или вне таблицы, если смена не была выбрана.
- Для статических файлов добавлена версия, чтобы браузер быстрее подтягивал свежий интерфейс после доработок.
- Визуальная подсветка выделенных ячеек сделана мягче.

## 2026-07-13

- Реализован этап 6: массовое заполнение ячеек графика.
- Добавлено выделение ячеек через `Ctrl`/`Cmd` + клик и `Shift` + клик.
- Выбор смены в любой выделенной ячейке сразу применяется ко всем выделенным ячейкам без отдельной кнопки `Заполнить выделенные`.
- Добавлен API `POST /api/schedule/bulk-fill`.
- Массовое заполнение сохраняется в `data/schedule_data.json`, обновляет таблицу без перезагрузки и пересчитывает часы затронутых сотрудников.
- Закреплено правило: `Праздник` применяется к выбранным датам для всех сотрудников графика, а не к отдельной ячейке сотрудника; это действует и при одиночном редактировании.
- Реализован этап 5: копирование графика между месяцами.
- Добавлена кнопка `Копировать из...` и модальное окно выбора месяца-источника, целевого месяца и подтверждения перезаписи.
- Добавлен API `POST /api/schedule/month/copy`.
- Копирование учитывает количество дней целевого месяца и праздники из производственного календаря; при недоступном календаре выводится предупреждение о ручной проверке праздников.
- Заголовки дней в таблице графика получили разное цветовое оформление: рабочие дни сине-серые, выходные мягко-зеленые.
- Добавлено удаление графика месяца с обязательным модальным подтверждением.
- Для непустого графика добавлен выбор: очистить заполненные смены или удалить месяц целиком.
- Пустым считается график без заполненных смен, кроме `Праздник` и `Отпуск`.
- Добавлен API `DELETE /api/schedule/month`.
- Провайдер производственного календаря по умолчанию заменен на `КонсультантПлюс`.
- Добавлен разбор страницы КонсультантПлюс вида `https://www.consultant.ru/law/ref/calendar/proizvodstvennye/{year}/`; дни `td.holiday` используются как официальные праздники.
- `isDayOff API` сохранен как резервный публичный провайдер производственного календаря.
- Добавлен разбор строкового ответа `isDayOff`: код `8` считается официальным праздником; обычные выходные не превращаются в `П`.
- Документация интеграции дополнена endpoint `https://isdayoff.ru/api/getdata` и параметрами запроса.
- Добавлена миграция старых настроек календаря: устаревший/неизвестный провайдер автоматически заменяется на `КонсультантПлюс`.
- Реализован этап 4: создание нового месячного графика из интерфейса.
- Добавлен API `POST /api/schedule/month` для создания нового месяца.
- Добавлена страница настроек `Интеграции` для производственного календаря: провайдер, URL API, токен, URL конфига из GitHub/ветки, таймаут.
- Создание месяца работает с fallback: если производственный календарь не настроен или недоступен, график создается с датами, днями недели и сотрудниками.
- Просмотр `.md`-документов переведен в пользовательскую HTML-страницу внутри АС.
- Для документов добавлена отдельная кнопка `Скачать`; открытие документа больше не инициирует загрузку файла.
- Добавлена страница документации в интерфейсе АС: `/docs`.
- В верхнюю навигацию добавлена ссылка `Документация`.
- Документы проекта теперь можно открыть из приложения: руководство пользователя, API, источник требований, правила графиков и журнал изменений.
- Закреплено правило сопровождения: при каждой функциональной доработке обновлять связанные `.md`-документы и этот журнал.
- Ранее добавлены и зафиксированы документы для эксплуатации АС: `docs/user-guide.md`, `docs/api.md`, `docs/ui-and-integrations-source.md`.
- Реализована работа с графиком после импорта без повторного чтения Excel: дальнейшие изменения сохраняются в `data/schedule_data.json`.
- Реализованы редактирование ячеек графика, добавление сотрудника в месяц и удаление сотрудника из текущего месяца.
