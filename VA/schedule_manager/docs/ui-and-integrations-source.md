# UI and Integrations Requirements Source

Основной источник требований в проекте: `docs/source/ui-and-integrations-guide.html`.

Исходный файл пользователя: `/Users/antonvaskin/Downloads/ui-and-integrations-guide.html`.

Дата фиксации в проекте: 2026-07-13.

При доработках АС использовать эти требования как приоритетные для архитектуры, UI и API.

Ключевые правила для текущего проекта:

- Новые действия строить цепочкой `страница -> JavaScript -> API route -> service -> state`.
- После загрузки Excel график живет в `data/schedule_data.json`; дальнейшая работа не должна перечитывать Excel.
- Для изменений графика использовать стабильный JSON-контракт `ok/data/error/meta`.
- Route отвечает за HTTP-вход и ответ, service отвечает за бизнес-сценарий и сохранение состояния.
- UI должен обрабатывать success, validation error, backend error и non-JSON response без падения страницы.
- Мутации графика должны сохраняться в состоянии АС и быть проверяемы тестами сервиса, API и smoke UI.
- Правила графика и будущее автозаполнение должны опираться на справочник сотрудников, локации и справочник компетенций, а не на хардкод фамилий или одиночное поле роли.
- Массовое заполнение обычных смен применяется только к выделенным ячейкам; массовое заполнение `Праздник`/`П` применяется к выбранным датам для всех сотрудников графика.
- Выделение ячеек графика должно поддерживать табличный сценарий: протягивание прямоугольной области, `Shift` для диапазона от якоря, `Ctrl`/`Cmd` для точечного добавления или добавления следующего диапазона протягиванием, `Escape` для сброса выделения.
- Управление выбранным графиком должно находиться в разделе `График`; блок `Состояние на сегодня` используется как компактный информер без управляющих кнопок и без ссылки на API.
- Заголовок раздела `График` должен показывать текущий месяц рядом с названием; состояние выделения ячеек располагается в строке выбора года и месяца.
- `Импорт` находится справа в верхнем заголовке окна; далее здесь же может появиться экспорт.
- Панель `Сегодня` должна быть компактным информационным информером, а не крупным блоком: дата, основной дежурный и короткие чипы дежурных смен; основная смена `8` не выводится.
- Выбор года и месяца графика должен быть компактным и применяться автоматически без кнопки `Показать`.
- Создание нового месяца должно сохранять график в `data/schedule_data.json` и не обращаться к Excel.
- Удаление месяца должно требовать подтверждения; непустой график нельзя удалить как пустой без явного выбора очистки или удаления целиком.
- Копирование месяца должно адаптировать данные к календарю целевого месяца, количеству дней и праздникам; при недоступном производственном календаре нужно показывать предупреждение о ручной проверке праздничных дней.
- Экспорт графика должен формировать новую Excel-книгу из текущих данных АС, а не перечитывать исходный Excel; обязательные видимые листы: сам график месяца, `К публикации`, `Для внесения в табель`.
- Интеграции с внешними источниками должны иметь отдельные настройки и безопасный fallback, если внешний сервис недоступен.
- Не выводить пользователю внутренние exception details, raw paths или технические traceback.

Перед началом новой доработки нужно сверяться с этим документом и применять его как более приоритетный источник, если старый план реализации противоречит требованиям.
