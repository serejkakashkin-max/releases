# Руководство пользователя АС графиков

## Назначение

АС управляет графиком дежурств после первичной загрузки Excel-файла. Excel используется только как источник импорта. После загрузки все изменения выполняются в данных АС, в `data/schedule_data.json`.

## Запуск

```bash
cd work/schedule_manager
source venv/bin/activate
python run.py
```

Открыть в браузере: `http://127.0.0.1:5055`.

Если сервер запущен вручную на другом порту, использовать адрес, который показан в консоли.

## Главная страница

На главной странице показаны:

- компактная панель состояния на сегодня;
- основной дежурный сейчас;
- состав дежурных смен на текущую дату без основной смены `8`;
- компактный блок управления графиком;
- фильтры по году и месяцу внутри раздела `График`;
- таблица графика.

Колонка `Часы` в таблице графика отображается целым числом. Область таблицы, ширина колонок и высота строк подстраиваются под размер окна браузера. Таблицы справочников сотрудников и смен используют тот же принцип адаптации.

Выбор года или месяца применяется автоматически после выбора значения. Кнопка `Показать` не используется.

Кнопка `Импорт` в правой части верхнего заголовка открывает окно управления загрузкой:

- загрузить новый Excel-файл;
- сбросить текущий график.

Кнопка `Создать график` открывает окно создания нового месяца.

Кнопка `Копировать из...` открывает окно копирования графика из другого месяца.

Кнопка `Экспорт` скачивает текущий график в Excel. В книге создаются три видимых листа: сам график выбранного месяца, `К публикации` и `Для внесения в табель`. Листы для публикации и табеля формируются из текущих данных АС, а не из исходного Excel-файла.

## Загрузка графика

1. Нажать `Импорт`.
2. Выбрать Excel-файл `.xlsx` или `.xls`.
3. Нажать `Загрузить график`.

После загрузки месячные графики переносятся во внутренние данные АС. Дальнейшая работа не перечитывает Excel.

## Редактирование ячейки

1. Нажать на ячейку графика.
2. Выбрать смену из списка.
3. Значение сохранится без перезагрузки страницы.
4. Часы сотрудника пересчитаются автоматически.

## Массовое заполнение ячеек

1. Выделить ячейки графика протягиванием мыши, `Ctrl`/`Cmd` + клик или `Shift` + клик.
2. Нажать на любую выделенную ячейку.
3. Выбрать смену или пустое значение в выпадающем списке.

АС сохранит изменения в `schedule_data.json`, обновит ячейки без перезагрузки страницы и пересчитает часы затронутых сотрудников.

Чтобы выделить несколько отдельных диапазонов, удерживать `Ctrl`/`Cmd` и протягивать следующий диапазон мышью.

`Escape` снимает текущее выделение.

Важно: `Праздник` относится к дате, а не к одному сотруднику. Если выбрать `Праздник` или `П`, АС проставит праздник на выбранные даты всем сотрудникам текущего графика. Это правило действует и при одиночном выборе смены в одной ячейке.

## Добавление сотрудника в месяц

1. Нажать `Добавить сотрудника` над таблицей графика.
2. Выбрать активного сотрудника.
3. Выбрать способ заполнения:
   - пустые ячейки;
   - `8` по рабочим дням.
4. Нажать `Добавить`.

Строка добавится в текущий месяц и сохранится в `schedule_data.json`.

## Удаление сотрудника из месяца

1. В строке сотрудника нажать `Удалить`.
2. Подтвердить удаление.

Удаляется только строка сотрудника в текущем месяце. Справочник сотрудников при этом не меняется.

## Удаление графика месяца

1. Открыть нужный год и месяц.
2. Нажать `Удалить график`.
3. Подтвердить действие в модальном окне.

Если в графике нет заполненных смен, кроме `Праздник` и `Отпуск`, АС предложит удалить месяц из `schedule_data.json`.

Если в графике есть заполненные смены, АС предложит:

- `Сделать пустым` — очистить рабочие смены и сохранить `Праздник`/`Отпуск`;
- `Удалить график` — удалить весь месяц целиком.

## Создание нового месяца

1. На главной странице нажать `Создать график`.
2. Выбрать год и месяц.
3. Выбрать источник сотрудников:
   - из последнего графика;
   - из справочника активных сотрудников.
4. Нажать `Создать`.

Новый месяц сохраняется в `data/schedule_data.json` и появляется в фильтрах года/месяца. Если производственный календарь не настроен или недоступен, АС создаст график только с датами, днями недели и сотрудниками.

## Копирование графика между месяцами

1. На главной странице нажать `Копировать из...`.
2. Выбрать месяц-источник.
3. Выбрать год и месяц назначения.
4. Если целевой месяц уже есть, включить `Перезаписать целевой месяц`.
5. Нажать `Скопировать`.

АС переносит строки сотрудников и смены по номерам дней, адаптируя график к количеству дней в целевом месяце. Праздники целевого месяца берутся из производственного календаря и проставляются как `П`.

Если производственный календарь недоступен, АС сохраняет скопированный график и показывает предупреждение: нужно свериться с производственным календарем и вручную проставить праздничные дни.

## Автопланирование пустого графика

Кнопка `Автоплан` появляется только для пустого графика. Пустым считается график без рабочих смен, кроме `Праздник` и `Отпуск`.

Перед запуском АС открывает модальное окно и требует подтвердить, что все отпуска и праздничные дни на планируемый месяц уже внесены. Если в месяце нет праздников, это тоже считается проверенным состоянием. Без подтверждения автопланирование не запускается.

Первая версия автопланировщика работает детерминированно:

- не читает исходный Excel после загрузки графика;
- назначает недельные дежурные смены по сохраненным данным АС;
- не меняет уже внесенные вручную смены и дозаполняет начатые недельные блоки тем же сотрудником;
- если месяц начинается внутри рабочей недели, продолжает недельные смены из предыдущего сохраненного графика;
- учитывает нагрузку за три предыдущих месяца по дежурным сменам;
- выравнивает количество дежурных блоков внутри планируемого месяца, чтобы один сотрудник не забирал несколько недель подряд при наличии свободных альтернатив;
- не повторяет одну и ту же московскую дежурную смену у одного сотрудника в пределах месяца, если есть альтернативы;
- разводит вечерние смены `ВД`/`ВР` по разным сотрудникам между соседними неделями, включая переход между месяцами, если есть альтернативы;
- при назначении вечерних смен `ВД`/`ВР` отдает приоритет сотрудникам с компетенцией `МПР-координатор`, но не ставит двух МПР-координаторов в вечерние смены одновременно;
- если МПР-координатор уже назначен в `ДД`, не ставит другого МПР-координатора в вечернюю смену этой же недели при наличии альтернатив;
- не ставит руководителя в недельные дежурные смены;
- назначает ровно одну `ВХ` на каждый выходной или праздничный день активным московским сотрудникам с включенным признаком `Готов к сверхурочке`;
- при назначении `ВХ` учитывает количество `ВХ` за три предыдущих месяца и в текущем месяце, а руководителя рассматривает с наименьшим приоритетом;
- не ставит сотрудника с компетенцией `Новичок` в основные дежурные смены `ДД`, `ВД` или `ХД`, если в его локации есть другие сотрудники;
- учитывает локацию Москва/Хабаровск;
- не назначает сотрудника в дежурную неделю, если на этой неделе у него есть отпуск;
- на каждый выходной или праздничный день назначает одного сотрудника в `ВХ`;
- после заполнения пересчитывает часы и запускает проверку правил.

После успешного запуска под таблицей появляется информационный блок `График сформирован автопланировщиком`. В нем указано, что график создан автопланом, когда это произошло и какие правила использовались при распределении смен.

В этом блоке есть раздел `Пояснения по назначениям`. Для каждой назначенной смены указано:

- сотрудник;
- смена;
- период;
- почему этот сотрудник подходил под смену;
- сколько было доступных кандидатов;
- какая нагрузка была у сотрудника до назначения.

Для дежурных смен в самой таблице также показываются аккуратные хинты при наведении на ячейку. Хинты не выводятся для `8`, `Праздник` и `Отпуск`, чтобы не перегружать график.

Если после этого вручную изменить любую смену, выполнить массовое заполнение, добавить или удалить сотрудника из графика, отметка автопланировщика очищается. Такой график считается уже скорректированным вручную.

## Настройки

Верхнее меню:

- `График` — главная страница с текущим графиком;
- `Сотрудники` — справочник сотрудников;
- `Компетенции` — справочник компетенций сотрудников;
- `Смены` — справочник смен.
- `Интеграции` — настройки внешних источников, включая производственный календарь.
- `Документация` — требования, руководство, API, правила графиков и журнал изменений.

Цвета, часы, сокращения и названия смен из справочника сразу используются при отображении и редактировании графика. Существующие смены редактируются прямо в строках таблицы. Новая смена добавляется через компактную кнопку `+` рядом с заголовком `Список смен`; поле `Сокр.` является обязательным и используется как краткое обозначение смены в графике.

В справочнике `Компетенции` ведутся признаки, которые можно назначать сотрудникам. Базовые системные компетенции:

- `Руководитель`;
- `Сотрудник сопровождения`;
- `МПР-координатор`.

Существующие компетенции редактируются прямо в строках таблицы. Новая компетенция добавляется через компактную кнопку `+` рядом с заголовком `Список компетенций`.

В справочнике сотрудников для каждого сотрудника задаются статус, локация и набор компетенций. Эти поля сразу используются при проверке графика:

- `Локация` определяет ограничения Москва/Хабаровск для смен;
- компетенция `Руководитель` ограничивает сотрудника сменой `8`;
- компетенция `МПР-координатор` включает правила совместимости МПР-координаторов в дневных и вечерних сменах.

Данные существующих сотрудников редактируются прямо в строке таблицы. Изменение ФИО, компетенций, email, телефона, статуса или локации сохраняется автоматически после выбора значения или выхода из измененного поля.

Новый сотрудник добавляется через компактную кнопку `+` рядом с заголовком `Список сотрудников`. Кнопка открывает модальное окно добавления, поэтому форма не занимает место над таблицей постоянно.

В карточке сотрудника компетенции выбираются в одном поле с множественным выбором. Выбранные значения отображаются чипами, список можно отфильтровать поиском или переключить на режим `Выбранные`.

У каждого сотрудника в данных АС есть явное поле компетенций. Если сотрудник загружен из Excel и его еще нет в справочнике, АС назначает базовую компетенцию `Сотрудник сопровождения`.

## Интеграции

В разделе `Интеграции` настраивается производственный календарь:

- включение или отключение внешнего календаря;
- провайдер;
- URL источника календаря;
- токен;
- URL конфигурации из GitHub/ветки или корпоративного репозитория;
- таймаут запроса.

По умолчанию используется `КонсультантПлюс`: `https://www.consultant.ru/law/ref/calendar/proizvodstvennye/`.

Для `КонсультантПлюс` АС открывает страницу года, например `https://www.consultant.ru/law/ref/calendar/proizvodstvennye/2027/`, читает таблицы календаря и берет дни с классом `holiday` как официальные праздники для отметки `П`.

Для `isDayOff API` АС запрашивает месяц с параметрами `year`, `month`, `cc=ru`, `pre=1`, `holiday=1` и получает строку по дням месяца. Код `8` считается официальным праздником и используется для отметки `П` в новом графике. Обычные выходные не помечаются как праздник.

Для режима `Другой JSON API` ожидается JSON с массивами `holidays` и `workdays`. Значения могут быть датами `YYYY-MM-DD` или числами дней месяца.

## Оперативное состояние

Состояние на сегодня доступно:

- на главной странице;
- через API: `GET /api/today`.

## Документация

Раздел `Документация` открывает документы прямо в окне АС. Для каждого документа доступно отдельное действие `Скачать`, если нужен исходный файл.
